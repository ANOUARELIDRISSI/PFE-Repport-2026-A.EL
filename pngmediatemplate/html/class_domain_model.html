Domain Data Model

The analytical domain is represented as evidence documents, semantic chunks, graph entities, relation assertions, structured records, and vector records.
Document
MongoDB source evidence
  • id, title, content
  • source_url, source_name
  • published_date, scraped_at
  • graph_extracted, structured_extracted
DocumentChunk
MongoDB retrieval unit
  • chunk_id, document_id
  • chunk_text, enriched_text
  • section_header, word_count
  • entity_ids link text to graph nodes
GeopoliticalEntity
Neo4j canonical node
  • id, name, canonical_name
  • POLITICIAN, PARTY, COUNTRY
  • ORGANIZATION, INSTITUTION, EVENT
  • MINISTRY, COMMITTEE, FOREIGN_ACTOR
RelationAssertion
Neo4j edge and MongoDB assertion
  • from_entity_id, to_entity_id
  • relation_type, confidence
  • evidence, source_doc_ids
  • valid_from, valid_to for temporal relations
StructuredRecord
MongoDB analytical collections
  • bills, votes, committees
  • contributions, conflict_flags
  • stored with document_id provenance
  • bills and donors are not graph node labels
VectorRecord
ZVec semantic index
  • chunk_id, embedding vector
  • content stores enriched text
  • metadata.document_id
  • metadata.entity_ids
