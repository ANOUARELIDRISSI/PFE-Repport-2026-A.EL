Evaluation Pipeline Architecture

The evaluation runner sends benchmark questions to the live backend, collects answers and evidence, applies judge-based metrics, then aggregates per-mode results.
Benchmark Dataset
benchmark/questions.jsonl
fourteen factual, relational, temporal, and comparative questions with ground truth.
Evaluation Runner
loads samples, selects rag, cot, or reflexive, and calls the live API.
Backend Query Path
retrieves chunks and graph facts, generates answer, returns sources, confidence, and trace data.
Mistral Judge
scores generated answers and retrieved contexts with structured judge prompts.
RAGAS-style metric family
Faithfulness
claims supported by context
Answer Relevancy
answer addresses question
Context Precision
retrieved chunks are useful
Context Recall
needed evidence is retrieved
Answer Correctness
alignment with ground truth
Evaluation Report
stores per-question scores, computes averages, and compares reasoning modes.
